# MentoHub (멘토허브) 🎓

> **대학생 멘토링 프로그램 통합 관리 서비스**  
> 멘토링 일정 조율, 학습 일지 기록, 보고서 수작업 작성의 비효율을 해결하기 위해 개발된 **서버리스(Client-Side Only) 통합 웹 플랫폼**입니다.

---

## 📖 프로젝트 개요 (Background)
기존의 대학생 멘토링 프로그램은 일정 관리(캘린더), 학습 일지(기록), 사무국 보고서 제출(A4 서식)이 각각 다른 플랫폼에서 파편화되어 이루어져 멘토와 멘티 모두 번거로움을 겪었습니다.  
**MentoHub**는 이 세 가지 프로세스를 단일 웹 페이지로 결합하여:
1. 멘토링 일정 수립 및 변경
2. 과목/일자별 상세 지도 내용 기록
3. 누적 데이터를 기반으로 한 보고서 자동 생성 및 PDF 다운로드
4. 멘토-멘티 간 질의응답 피드

위 모든 작업을 복잡한 서버 구축 없이 브라우저 환경에서 쉽고 세련되게 처리할 수 있도록 개발되었습니다.

## ✨ 주요 기능 (Key Features)

### 1. 역할군 기반 권한 관리 & 원클릭 모의 로그인
- **멘토(김민수)**, **멘티(이영희)**, **운영 관리자(사무국)**로 구분된 3종의 사용자 계정을 지원합니다.
- 사이트 최초 접속 시 즉시 테스트가 가능한 모의 로그인 화면을 제공하며, 메인 화면 우측 상단의 **역할 변경 드롭다운**을 통해 로그인 세션을 즉시 바꾸며 권한 검증을 할 수 있습니다.
  - *멘토*: 일정 추가/수정/삭제, 학습 기록 등록/수정/삭제, 피드 작성, 보고서 생성 및 의견 편집 권한 전체 소유.
  - *멘티*: 일정 및 학습 기록은 읽기 전용, Q&A 피드 및 댓글 작성 권한 소유.
  - *운영 관리자*: 전체 멘토링 한눈에 확인 및 보고서 점검 전용 권한 소유.

### 2. 종합 현황 대시보드 (Dashboard)
- 멘토링 누적 시간, 완료된 세션 횟수, 등록된 학습 기록 건수의 통계를 시각적 카드로 제공합니다.
- 다음 멘토링 예정 일정에 대한 **D-Day 디스플레이** 기능이 활성화됩니다.
- 일정 수립, 기록 작성, 피드 등록 등 최근에 일어난 활동을 시간 순으로 정리한 **최근 활동 타임라인**을 제공합니다.

### 3. 커스텀 인터랙티브 캘린더 (Calendar)
- Vanilla JS로 구현한 반응형 월별 달력 컴포넌트입니다.
- 일정 상태(예정: Scheduled / 완료: Completed)에 따라 색상이 구분된 이벤트 배지가 달력 내부에 직접 표기됩니다.
- 멘토 권한일 경우 달력 셀을 더블 클릭하여 새 일정을 바로 추가하고, 배지를 클릭해 일정을 수정 또는 삭제하는 인터랙티브 모달이 실행됩니다.

### 4. 과목별 학습 기록부 (Study Logs)
- 날짜, 과목(수학, 과학, 영어, 코딩, 기타), 학습 시간, 주제, 상세 지도시항을 입력 폼을 통해 기록합니다.
- 입력된 내용은 세련된 이력 테이블 형태로 나열되며, 학습 달성도/이해도(%)를 시각적 프로그레스 바로 한눈에 확인할 수 있습니다.

### 5. SNS 스타일 소통 피드 (Q&A Feed)
- 멘토와 멘티가 실시간으로 질문을 올리고 의견을 주고받을 수 있는 타임라인 피드입니다.
- 글마다 댓글/답변 쓰기 기능이 지원되어 과제 피드백, 일정 조율 등 일상적인 소통 창구로 사용됩니다.

### 6. 활동 보고서 자동 생성 & PDF 다운로드
- 보고서 탭에서 원하는 **집계 시작일**과 **종료일**을 선택하고 데이터 집계를 실행합니다.
- 해당 기간 내 수행된 캘린더 일정이 회기순으로 자동 나열되며, 각 일자에 연동된 학습 기록지의 내용이 표 우측에 자동으로 바인딩됩니다.
- 이달의 총 활동 횟수와 누적 시간이 자동 합산되어 서식에 입력됩니다.
- 사용자가 직접 타이핑하여 내용을 가감할 수 있는 **편집 가능한 종합 의견 텍스트 영역**을 내장하였습니다.
- **[PDF 다운로드]** 클릭 시, `html2pdf.js`가 A4 용지 비율로 스타일링된 보고서 컨테이너를 스캔하여 깨짐 없는 고품질 PDF 벡터 파일로 변환하여 기기에 즉시 저장합니다.

### 7. 프리미엄 CSS 테마 시스템
- 딥 인디고 및 퍼플 그라데이션 포인트가 가미된 트렌디한 SaaS 스타일의 디자인을 갖추고 있습니다.
- 사이드바 하단 버튼을 통해 **다크 모드와 라이트 모드 전환**을 전역 지원합니다.

## 🛠 기술 스택 (Tech Stack)
- **Frontend Core**: Vanilla HTML5, Vanilla CSS3 (CSS Variables, Flexbox/Grid Layout), Vanilla JavaScript (ES6+)
- **Icons**: FontAwesome CDN (`v6.4.0`)
- **Fonts**: Google Fonts (Outfit, Inter, Noto Sans KR)
- **Libraries**: html2pdf.js (`v0.10.1` via CDN)
- **Local Server**: Node.js `http-server`

## 🚀 로컬 실행 방법 (How to Run)

본 프로젝트는 순수 정적 브라우저 자원으로 구성되어 있어 빌드가 필요하지 않으며 오프라인 구동도 가능합니다. 로컬 서버 환경에서 실행하여 온전히 확인하시려면 다음의 안내를 따릅니다.

1. **저장소 클론**
   ```bash
   git clone <your-repository-url>
   cd <repository-directory>
   ```

2. **로컬 개발 서버 기동**  
   Node.js가 설치된 환경에서 아래 명령어를 실행하면 `npx`를 통해 경량 로컬 서버가 백그라운드로 실행됩니다.
   ```bash
   npm start
   ```

3. **웹 접속**  
   서버가 구동되면 웹 브라우저를 켜고 다음 주소로 이동합니다.
   ```text
   http://localhost:3000
   ```

## 🌐 GitHub Pages 배포 방법 (Deployment)

MentoHub는 100% 프론트엔드 정적 파일(Vanilla HTML, CSS, JS)로만 구성되어 있어, 빌드 과정 없이 GitHub Pages를 통해 무료로 손쉽게 호스팅 및 실배포를 할 수 있습니다.

### 방법 1: GitHub 리포지토리 설정에서 배포 (간편 설정)
1. 개발한 소스 코드를 GitHub 원격 저장소의 `main` 브랜치에 푸시(Push)합니다.
2. 해당 GitHub 저장소의 우측 상단 **[Settings]** 탭으로 이동합니다.
3. 좌측 사이드바 메뉴에서 **[Pages]** 메뉴를 클릭합니다.
4. **Build and deployment** 항목의 **Source**에서 **Deploy from a branch**를 선택합니다.
5. 바로 아래 **Branch**에서 **`main`** 브랜치와 루트 디렉토리 **`/ (root)`**를 선택한 후 **[Save]** 버튼을 누릅니다.
6. 약 1~2분 후 상단에 발급되는 URL(예: `https://<username>.github.io/<repository-name>/`)로 접속하면 실제 구동되는 서비스를 확인할 수 있습니다.

### 방법 2: GitHub Actions를 통한 배포 자동화 (워크플로우 파일 포함)
- 본 저장소의 [.github/workflows/deploy.yml](file:///.github/workflows/deploy.yml) 설정에 따라 GitHub Actions 빌드/배포 자동화가 지원됩니다.
- 저장소 **[Settings] -> [Pages]** 탭에서 **Source**를 **GitHub Actions**로 변경하면, 향후 코드를 `main` 브랜치에 푸시할 때마다 배포 프로세스가 즉시 실행됩니다.
